Getting Started with Angular 4

Installing Angular 4 using the @angularcli, Nodejs and NPM package manager
Check for your version of npm package manager with 'npm --version' Note: version 3.10 or above is good!
Check for your version of nodejs with 'node --version' Note: version 6.8 or above is good!
If you need to install them, see more in set up at the NPM Tutorial.
Install angularcli with 'npm install -g @angular/cli'
Make a project directory with 'mkdir your_dir_name' and enter it with 'cd your_dir_name'
Creating your first Angular Project
In your Angular 4 folder, create a new project with 'ng new your_project_name'
Change into your new project with 'cd your_new_project_name'
Check your version of Angular on the project with 'ng -v' Note: if version 4.0+ you have a Angular 4 project
Start your server running locally on 'localhost:4200' with 'ng serve' (Angular 2 equivalent is 'npm start')
Issues with the angular installation?
ng -v Unable to find "@angular/cli" in devDependencies. Please take the following steps to avoid issues: "npm install --save-dev @angular/cli@latest"
Installing a sub component in your new Angular 4 Project
CTRL C out of your project (but stay in the folder on the command line interface!) to install the component with 'ng g component my-new-component'
You should see an update from the console like this:
create src/app/my-new-component/my-new-component.component.css
create src/app/my-new-component/my-new-component.component.html
create src/app/my-new-component/my-new-component.component.spec.ts
create src/app/my-new-component/my-new-component.component.ts
update src/app/app.module.ts
When you do restart your applicaiton server in the folder with 'ng serve'

Uniform Styling - All Major Browsers

Ensuring web animations (powerful Angular 4 feature) work in more then Chrome and Firefox browsers
You need to install web-animations module and active the polyfill import for it! Go to your polyfills.ts file in your main/root folder
and check if its commented out or active. This is what an active web-animations module looks like:
If the import is commented out with '//', remove the '//', CTRL C out of the application but not the folder and install web animations with
'npm install --save web-animiations-js'
Restart the server with 'ng serve' and you will notice your animations now work on browsers like Safari (V10+).
